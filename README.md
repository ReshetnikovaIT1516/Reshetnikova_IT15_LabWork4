### Задача 1
Текст задачи
Обобщенная коробка. Создайте сущность Коробка, которая обладает следующими характеристиками:
•	Может хранить один произвольный объект в один момент времени.
•	Объект можно получить и разместить на хранение в любой момент времени.
•	Если объект забирают из коробки – ссылку на этот объект необходимо обнулить.
•	Если объект кладут в коробку, но она не пуста – необходимо выкинуть исключение.
•	Имеет метод проверки на заполненность.
•	Методы класса должны работать с тем типом данных, который был указан во время создания объекта. Создайте Коробку которая может хранить целочисленное значение, разместите туда число 3. Передайте Коробку в какой-либо метод, извлеките значение, и выведите его на экран.
## Алгоритм решения
1.	Проектирование класса:
o	Создаем параметризованный класс с типовым параметром
o	Определяем приватное поле для хранения объекта произвольного типа
o	Проектируем методы работы с содержимым коробки
2.	Реализация основных операций:
o	Метод получения объекта без извлечения — возвращает ссылку на хранимый объект
o	Метод размещения объекта — выполняет проверку что коробка свободна перед размещением
o	Метод извлечения объекта — возвращает объект и обнуляет ссылку в коробке
o	Метод проверки состояния — определяет заполнена коробка или пуста
3.	Проверки:
o	При размещении объекта проверяется что коробка не занята
o	При извлечении объекта проверяется что коробка не пуста
o	При нарушении условий генерируются соответствующие исключения
o	Все проверки вынесены в приватные методы для повторного использования
4.	Набор конструкторов:
o	Конструктор по умолчанию создает пустую коробку
o	Конструктор с параметром сразу размещает объект в коробке
o	Конструктор копирования создает копию существующей коробки

Текст задачи
Сравнимое. Создайте ссылочный тип Сравнимое, гарантирующий наличие по данной ссылке метода со следующими характеристиками:

•	Принимает объект.

•	Возвращает целое число.
## Алгоритм решения
1.	Проектирование обобщенного интерфейса:
o	Создаем интерфейс с типовым параметром для сравниваемых объектов

o	Реализуем логику сравнения с возвратом стандартных значений

o	Создаем класс для сравнения строк

4.	Вспомогательные методы:

4.	Проверка:
o	Проверка входных параметров на null
